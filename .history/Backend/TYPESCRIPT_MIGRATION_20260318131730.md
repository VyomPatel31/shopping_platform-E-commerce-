# Backend TypeScript Migration Guide

## Overview

The entire Backend has been converted from JavaScript to TypeScript for better type safety, maintainability, and developer experience.

## Changes Made

### 1. Project Configuration

#### Updated `package.json`
- Changed `"type": "module"` for ES modules
- Updated scripts:
  - `build`: Compiles TypeScript to JavaScript
  - `dev`: Runs with nodemon for hot reload
  - `start`: Runs compiled JavaScript from `dist/`
- Added TypeScript-related devDependencies

#### Created `tsconfig.json`
- Target: ES2020
- Module: ES2020
- Output directory: `dist/`
- Source directory: `src/`
- Strict mode enabled
- ESM support enabled

#### Created `nodemon.json`
- Watches `src/` directory
- Executes on `.ts` file changes
- Uses ts-node for running

### 4. Build Process

**Development:**
```bash
npm run dev
# Uses ts-node + nodemon for hot reload
```

**Production:**
```bash
npm run build  # Compiles src/ to dist/
npm start      # Runs dist/server.js
```

### 5. Migration Steps

If you already have code using the old JavaScript files:

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Create .env file:**
   ```bash
   cp .env.example .env
   ```

3. **Build once:**
   ```bash
   npm run build
   ```

4. **Run development:**
   ```bash
   npm run dev
   ```

5. **Deploy production:**
   ```bash
   npm run build
   npm start
   ```

### 6. Key TypeScript Features

#### Request Type Augmentation
```typescript
declare global {
  namespace Express {
    interface Request {
      user?: { _id: string; role: string; ... }
      sessionID?: string
    }
  }
}
```

#### Generic Response Builder
```typescript
const buildResponse = <T = any>(code: number, response: T): ApiResponse<T>
```

#### Model Interfaces
```typescript
export interface IUser extends Document {
  name: string
  email: string
  role: 'user' | 'admin'
  // ...
}
```

#### Typed Controllers
```typescript
export const createOrderController = async (
  req: Request, 
  res: Response
): Promise<void> => {
  // Implementation
}
```

### 7. API Response Format

All endpoints follow this format:

**Success:**
```json
{
  "success": true,
  "code": 200,
  "response": { /* data */ }
}
```

**Error:**
```json
{
  "success": false,
  "code": 400,
  "message": "Error message"
}
```
